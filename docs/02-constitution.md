# Desklabs Constitution

Version: 1.0  
Status: Active

---

# Purpose

The Constitution defines the non-negotiable principles that guide every product, design, and engineering decision inside Desklabs.

Whenever a new feature, workflow, or design is proposed, it must be evaluated against this Constitution before implementation.

The Constitution changes rarely.

Implementation changes often.

---

# Product DNA

These are the core beliefs of Desklabs.

They are not optional.

They are not features.

They define the identity of the product.

## Conversation First

Every business process starts from a conversation.

Conversation is the primary object of Desklabs.

## Context Never Lost

Users should never search for information that the system already knows.

Context follows the user everywhere.

## Workspace Over Modules

Users think in workspaces.

Not database entities.

## Calm Workspace

Reduce noise before adding capability.

Focus is a feature.

## Human Before AI

Artificial Intelligence assists people.

It never replaces human judgement.

---

# Product Principles

## Work Before Data

Users come to finish work.

Not to manage records.

## Context Before Navigation

Show information.

Do not force navigation.

## Progressive Disclosure

Complexity appears only when necessary.

## One Primary Action

Every screen has one dominant action.

## Every Screen Answers One Question

Each workspace exists to answer one primary question.

## Every Pixel Earns Its Place

Every visible element must justify its existence.

---

# Non-Negotiables

Desklabs will never:

- Build features because competitors have them.
- Add dashboards full of charts without clear user value.
- Make AI the center of the interface.
- Require users to repeatedly enter the same information.
- Create workflows that increase cognitive load.

---

# Decision Framework

Every product decision must answer these questions.

1. Does it shorten the user's workflow?
2. Does it reduce cognitive load?
3. Does it preserve context?
4. Does it support the Calm Workspace philosophy?
5. Does it strengthen Conversation First?

If one answer is "No", the proposal must be reviewed again.

---

# AI Constitution

Artificial Intelligence exists to reduce work.

AI should:

- summarize
- recommend
- remind
- organize
- automate repetitive tasks

AI should never become the primary interface.

Users should feel the benefit.

Not the technology.

---

# UX Constitution

The interface should feel calm.

Navigation should be predictable.

Interactions should be obvious.

The system should explain itself through design instead of documentation.

Less interface.

More clarity.

---

# Engineering Constitution

Prefer simplicity over cleverness.

Prefer maintainability over shortcuts.

Prefer reusable systems over page-specific implementations.

Every implementation should improve the overall architecture.

---

# Product Oath

We do not build software to impress.

We build software to reduce work.

Every release should make the product feel lighter.

Every feature should make work easier.

Every decision should move the customer closer to a successful outcome.

If it does not improve the user's work, it does not belong in Desklabs.
