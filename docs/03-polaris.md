# Polaris Design Language

Version: 1.0
Status: Active

---

# Purpose

Polaris is the design language of Desklabs.

It ensures every workspace feels like one product.

Polaris is not a UI kit.

It is a system of decisions.

---

# Design Philosophy

Calm.

Focused.

Contextual.

Fast.

Human.

Every interface should reduce cognitive load.

Never increase it.

---

# Visual Principles

## Calm Workspace

Reduce visual noise.

Remove unnecessary borders.

Remove unnecessary colors.

Whitespace is part of the interface.

---

## Progressive Disclosure

Show only what users need now.

Advanced controls appear only when required.

---

## Consistency

The same action should always look and behave the same.

Never redesign components differently between pages.

---

## Hierarchy

Every screen has:

- One primary action
- Few secondary actions
- Everything else stays in the background

---

# Layout

Workspace Shell consists of:

- Sidebar
- Header
- Main Workspace
- Optional Context Panel

Every workspace uses the same shell.

---

# Sidebar

Purpose:

Select a workspace.

Not browse modules.

Rules:

- Fixed width
- One active workspace
- Icons are secondary
- Labels are primary
- Minimal visual noise

---

# Header

Purpose:

Orient the user.

Header should answer:

"Where am I?"

Avoid placing dashboards or large KPI cards here.

---

# Main Workspace

The primary working area.

Everything important happens here.

Never compete with side panels.

---

# Context Panel

Optional.

Collapsed by default.

Contains supporting information such as:

- Customer Passport
- AI Summary
- Workflow
- Travel Information

Context supports work.

It never replaces work.

---

# Color Principles

Color communicates meaning.

Never decoration.

Primary color is reserved for primary actions.

Success, Warning, and Error are used sparingly.

Avoid colorful interfaces.

---

# Typography

Typography communicates hierarchy.

Not decoration.

Rules:

- Clear headings
- Comfortable reading
- Consistent spacing
- Minimal font variations

---

# Components

Components must be reusable.

No page-specific buttons.

No duplicated cards.

Every new component should be reusable across workspaces.

---

# Motion

Motion should explain.

Not entertain.

Animations should:

- communicate changes
- preserve context
- reduce confusion

Avoid excessive animation.

---

# AI Experience

AI should feel invisible.

Good AI:

- summarizes
- recommends
- reminds

Bad AI:

- interrupts
- blocks work
- asks unnecessary questions

---

# Workspace Principles

Every workspace answers exactly one question.

Today

"What should I focus on today?"

Inbox

"Who needs my attention?"

Customers

"Who is this customer?"

Sales

"What opportunity should I move forward?"

Operations

"What departure needs attention?"

Finance

"What payment needs action?"

---

# Interaction Principles

Prefer:

One click.

One decision.

One outcome.

Reduce unnecessary navigation.

Keep users in context.

---

# Accessibility

Every interaction should be usable with:

- keyboard
- screen reader
- sufficient contrast
- clear focus states

Accessibility is part of quality.

Not an optional feature.

---

# Definition of Good Design

Good design feels obvious.

Users should remember the work they completed.

Not the interface they used.

If users notice the interface too much,

it is probably too complicated.
