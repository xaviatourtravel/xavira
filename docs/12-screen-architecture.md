# Desklabs Screen Architecture

Version: 1.0

---

# Philosophy

Every screen in Desklabs answers one question.

"What should I do next?"

If a screen cannot answer that question,

it should be redesigned.

---

# Standard Screen Structure

Every workspace follows the same structure.

────────────────────────

Topbar

↓

Screen Header

↓

Primary Work

↓

Secondary Work

↓

Supporting Information

────────────────────────

Never change this order.

---

# 01 Topbar

Contains

Global Search

Create Button

Notifications

Workspace Switcher

Profile

The topbar never changes.

---

# 02 Screen Header

Contains

Page Title

One sentence description

Optional Date

Never place metrics here.

---

Example

Today

AI already prepared today's work.

---

# 03 Primary Work

The first visible section.

Contains only one objective.

Examples

Start Here

Current Conversation

Customer Passport

Booking Summary

Task Queue

This section owns the page.

---

# 04 Secondary Work

Supports the primary work.

Examples

Queue

History

Participants

Payments

Upcoming Tasks

Maximum

Two sections.

---

# 05 Supporting Information

Everything else.

Examples

AI Summary

Workspace Health

Notes

Timeline

Insights

Should never compete visually with Primary Work.

---

# CTA Hierarchy

One Primary Button

Ghost Buttons

Links

Never multiple primary buttons.

---

# Information Hierarchy

Object

↓

Action

↓

Context

↓

Metadata

Never reverse this order.

---

# Scrolling

The first viewport should answer:

What should I do first?

Scrolling is for details,

never for orientation.

---

# Empty States

Every screen should still feel useful,

even with zero data.

Explain

Why

What next

Provide action.

---

# Workspace Checklist

Before shipping a screen ask:

✓ Is there one obvious starting point?

✓ Is there only one primary action?

✓ Can users understand this page within five seconds?

✓ Does the first viewport answer what should happen next?

✓ Is supporting information visually quieter?

✓ Is the layout consistent with every other workspace?

If any answer is "No",

the screen is not finished.
