# Copywriting Guide

**Status:** Stable\
**Version:** 1.0\
**Owner:** Product Design\
**Last Updated:** 2026-07-06\
**Applies To:** Entire Desklabs Platform

------------------------------------------------------------------------

# Purpose

Copy is part of the product experience.

Good interface copy reduces uncertainty, builds confidence, and helps
users make decisions faster.

------------------------------------------------------------------------

# Writing Principles

1.  Write for humans, not systems.
2.  Explain outcomes, not implementation.
3.  Prefer familiar language over technical jargon.
4.  Be concise.
5.  Every label should answer a user question.

------------------------------------------------------------------------

# Tone of Voice

Desklabs should feel:

-   Calm
-   Professional
-   Helpful
-   Confident
-   Transparent

Avoid sounding robotic, overly enthusiastic, or overly technical.

------------------------------------------------------------------------

# Preferred Vocabulary

  Avoid                  Prefer
  ---------------------- -----------------------
  Lead Qualification     Information Collected
  AI Insight             What You Should Know
  Files                  Resources
  AI Generated Summary   Conversation Summary
  Execute                Run
  Error                  Something went wrong
  Human Takeover         Hand Over to Team

Always choose terms that match how travel teams naturally speak.

------------------------------------------------------------------------

# Button Labels

Use verbs.

Good:

-   Send
-   Save
-   Publish
-   Continue
-   Assign
-   Open
-   Review

Avoid vague actions:

-   Submit
-   Execute
-   Process

------------------------------------------------------------------------

# Empty States

Structure:

1.  Title
2.  Explanation
3.  Primary action

Example:

Title: No resources yet.

Description: Recommended documents will appear here when they're
relevant.

CTA: Upload Resource

------------------------------------------------------------------------

# Errors

Explain:

-   What happened
-   Why it happened (if known)
-   What the user can do next

Avoid exposing technical details.

------------------------------------------------------------------------

# Success Messages

Keep them short.

Examples:

-   Saved.
-   Published successfully.
-   Customer assigned.
-   Reply copied.

Avoid unnecessary celebration.

------------------------------------------------------------------------

# AI Language

AI should never say:

-   As an AI...
-   I analyzed...
-   Based on my model...

Instead:

-   Here's a suggested reply.
-   I couldn't find information about that yet.
-   This recommendation is based on your Business Brain.

------------------------------------------------------------------------

# Labels

Prefer nouns.

Examples:

-   Customer
-   Booking
-   Resources
-   Timeline
-   Payments

Avoid internal engineering terminology.

------------------------------------------------------------------------

# Dates & Time

Use localized formatting.

Respect the selected application language.

------------------------------------------------------------------------

# Localization

All user-facing strings must support:

-   Bahasa Indonesia
-   English

Never hardcode visible text.

------------------------------------------------------------------------

# Anti-Patterns

Avoid:

-   Marketing language in workflows
-   Technical jargon
-   Long button labels
-   Multiple exclamation marks
-   Emoji in enterprise interfaces

------------------------------------------------------------------------

# Evolution

New terminology should be reviewed before becoming part of the shared
product vocabulary.
